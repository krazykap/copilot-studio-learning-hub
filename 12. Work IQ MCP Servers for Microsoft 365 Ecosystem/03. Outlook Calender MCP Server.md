# 03. Outlook Calender MCP Server - Enabling Intelligent Scheduling & Time Orchestration in Work IQ
Outlook Calendar MCP Server is a powerful component of the Work IQ platform that enables intelligent scheduling and time orchestration for users. By integrating with Microsoft 365, it allows users to seamlessly manage their calendars, schedule meetings, and optimize their time effectively.

## 🚀 Why Outlook Calendar MCP Server Matters
Scheduling is one of the most time-consuming and fragmented workflows in enterprises:
- Endless back-and-forth for meeting slots
- Lack of visibility into participant availability
- Time zone complexities across global teams
- Inefficient calendar utilization

Despite modern tools, coordination still relies heavily on **manual effort and human judgment**.
<pre><code>The <b>Outlook Calendar MCP Server</b> transforms scheduling into an <b>intelligent, automated, and context-aware process</b> within Work IQ.</code></pre>

## 🧠 What is the Outlook Calendar MCP Server?
The Outlook Calendar MCP Server is a domain-specific MCP implementation that exposes calendar operations as structured, AI-invokable tools.

It allows copilots to:
- Understand availability and scheduling constraints
- Schedules and manage meetings on behalf of users
- Resolve scheduling conflicts and optimize calendar utilization
- Optimize time management by suggesting best meeting times based on user preferences and historical data

### 🔑 Core Idea
<pre><code>Conver calendar operations into <b>intelligent scheduling</b> by providing a structured interface to calendar data and operations, allowing AI copilots to automate and optimize the scheduling process.</code></pre>

## ⚙️ Core Capabilities
The Outlook Calendar MCP Server provides a range of capabilities that enable intelligent scheduling:

### Calendar Retrieval
Capable of retrieving calendar data, including:
- Fetching calendar events and availability for users or groups
- Retriving daily, weekly, or monthly schedules
- Access meeting details such as attendees, locations, and time zones

## Free/Busy Detection
The server can analyze calendar data to determine free/busy status, allowing it to:
- Identify available time slots for scheduling meetings
- Detect scheduling conflicts and suggest alternative times
- Support multi-user scheduling by aggregating availability across participants

## Meeting Management
The server can manage meetings on behalf of users, including:
- Creating, updating, and canceling meetings
- Add agendas, notes, location, and attachments to meetings
- Handle recurring meetings and exceptions
- Reschedule meetings based on participant availability and preferences
- Update meeting details and notify participants of changes

## Time-Zone Intelligence
The server can handle time zone complexities by:
- Automatically converting meeting times to participants' local time zones

## 🏗️ Architecture Overview
The Outlook Calendar MCP Server is built on top of the Microsoft Graph API, which provides access to calendar data in Microsoft 365. It exposes a set of structured APIs that allow AI copilots to interact with calendar data and perform scheduling operations.

## 🔄 End-to-End Flow
🧑‍💼 Scenario:
<pre><code>Schedule a 30-minute meeting with the product team next week.</code></pre>

### Execution Steps:
- Copilot receives the scheduling request and identifies the need to access calendar data.
- Work IQ extracts intent:
	- Meeting duration: 30 minutes
	- Participants: Product team
- Orcherstrator invokes Outlook Calendar MCP Server to fetch availability for product team members.
- Calendar MCP Server retrieves calendar data and identifies free time slots via Graph API.
- Best meeting time is determined based on availability and user preferences.
- Copilot suggests the optimal meeting time to the user and requests confirmation.
- MCP invokes the Calendar MCP Server to create the meeting on behalf of the user.
- Meeting is created, and all participants are notified with the meeting details.

## 🧰 Tool Definitions

The Outlook Calendar MCP Server provides a comprehensive set of tools organized into functional categories for intelligent calendar management:

### 📋 Event Retrieval & Discovery

| Tool | Description |
|---|---|
| `ListEvents` | Retrieve a list of events for the user with given criteria such as start and end datetimes, meeting title, and attendee emails. Returns only the master event for recurring meetings. |
| `ListCalendarView` | Retrieve events from a user's calendar view with recurring events expanded into individual instances. Use this to find user meetings in general, or specific instances of recurring events within a time window before modifying, updating, or canceling. |
| `FindMeetingTimes` | Find meeting times that work for all attendees by analyzing availability. Suggests optimal meeting times based on organizer and attendee availability. Supports names or email addresses for attendees (names are automatically resolved to emails). |

### ✏️ Event Management & Modification

| Tool | Description |
|---|---|
| `CreateEvent` | Create a new calendar event with support for attendees, location, and time. Can provide names or email addresses for attendees (names resolved to emails automatically). Default duration is 30 minutes. All events automatically include a Teams meeting link. Supports recurring events. |
| `UpdateEvent` | Update an existing calendar event to add or remove attendees, modify details, and change timing. Names and email addresses are automatically resolved. **IMPORTANT**: When updating the body of an online meeting, retrieve the current event body first, extract the 'Microsoft Teams meeting' section, and append it to the new body to preserve meeting links. |
| `DeleteEventById` | Delete a calendar event by ID. Retrieve the event details first to extract the event ID before deletion. |
| `CancelEvent` | Cancel a calendar event and notify all attendees. Only the event organizer can cancel an event. Sends cancellation notifications to all participants. |

### ✅ Event Response & Acceptance

| Tool | Description |
|---|---|
| `AcceptEvent` | Accept a calendar event invitation. Optionally include a comment with the acceptance response. |
| `TentativelyAcceptEvent` | Tentatively accept a calendar event invitation (mark as tentative). Optionally include a comment with the tentative response. |
| `DeclineEvent` | Decline a calendar event invitation. Optionally include a comment with the decline response. |
| `ForwardEvent` | Forward a calendar event to other recipients. Can provide names or email addresses for recipients (names are automatically resolved to emails). |

### ⚙️ User & Resource Management

| Tool | Description |
|---|---|
| `GetUserDateAndTimeZoneSettings` | Retrieve date and timezone settings for a user including time zone, date format, time format, working hours, and language preferences. Can provide name, email address, or use 'me' for the current user. |
| `GetRooms` | Retrieve all meeting rooms defined in the user's tenant. Returns room names and email addresses for resource booking and availability checking. |

## 🔌 Integration with Microsoft Graph
Calendar MCP Server is built on top of the Microsoft Graph API, which provides access to calendar data in Microsoft 365. It uses Graph API endpoints to perform operations such as retrieving calendar events, checking availability, and managing meetings.

### 📍 Key Graph API Endpoints

The following table maps common Calendar MCP tools to their corresponding Microsoft Graph REST API endpoints:

| MCP Tool | HTTP Method | Graph API Endpoint | Description |
|---|---|---|---|
| `ListEvents` | GET | `/me/events` | Retrieve all events for the signed-in user |
| `ListEvents` (filtered) | GET | `/me/events?$filter=start/dateTime ge '{startDateTime}' and end/dateTime le '{endDateTime}'` | Retrieve events within a specific date/time range |
| `ListCalendarView` | GET | `/me/calendarview?startDateTime={start}&endDateTime={end}` | Retrieve calendar view with expanded recurring events |
| `CreateEvent` | POST | `/me/events` | Create a new calendar event |
| `UpdateEvent` | PATCH | `/me/events/{eventId}` | Update an existing calendar event |
| `DeleteEventById` | DELETE | `/me/events/{eventId}` | Delete a calendar event |
| `CancelEvent` | POST | `/me/events/{eventId}/cancel` | Cancel a calendar event and notify attendees |
| `FindMeetingTimes` | POST | `/me/calendar/getSchedule` | Find meeting times based on attendee availability |
| `AcceptEvent` | POST | `/me/events/{eventId}/accept` | Accept a calendar event invitation |
| `TentativelyAcceptEvent` | POST | `/me/events/{eventId}/tentativelyAccept` | Tentatively accept a calendar event |
| `DeclineEvent` | POST | `/me/events/{eventId}/decline` | Decline a calendar event invitation |
| `ForwardEvent` | POST | `/me/events/{eventId}/forward` | Forward a calendar event to other recipients |
| `GetUserDateAndTimeZoneSettings` | GET | `/me/mailboxSettings` | Retrieve user's date format, time format, and timezone settings |
| `GetRooms` | GET | `/me/findRooms` | Retrieve list of meeting rooms in the tenant |
| `GetRooms` (by mailbox) | GET | `/me/findRoomLists` | Retrieve all room lists in the organization |

### 🔐 Authentication Flow

All requests to the Microsoft Graph API must include a valid OAuth 2.0 access token in the `Authorization` header:

```
GET /me/events HTTP/1.1
Host: graph.microsoft.com
Authorization: Bearer {access_token}
```

**Token acquisition:**
1. Client registers application in Azure AD (Microsoft Entra ID)
2. User grants consent for required scopes (Calendars.Read, Calendars.ReadWrite, etc.)
3. Application receives refresh token and access token
4. Access token is included in every Graph API call
5. Token is refreshed before expiration using refresh token

### 🔐 Required Scopes

Calendar operations require appropriate Microsoft Graph scopes:
- `Calendars.Read` - for reading calendar events
- `Calendars.ReadWrite` - for creating, updating, and deleting events
- `Calendars.Read.Shared` - for reading shared calendars
- `Calendars.ReadWrite.Shared` - for modifying shared calendars

### ✨ Best Practice Tips

- **Specific Attendees**: Always provide names or email addresses for accurate availability checks
- **Duration Clarity**: Specify exact meeting duration (15-min, 30-min, 1-hour) for precise scheduling
- **Timezone Awareness**: Always mention time zones when scheduling across regions
- **Lead Time**: Include timeframe information ("next week", "ASAP", "by end of month")
- **Recurring Pattern**: Clearly define frequency for recurring meetings (weekly, bi-weekly, monthly)
- **Room Requirements**: Specify meeting room needs, video conference setup, or in-person location
- **Attendee Roles**: Clarify required vs. optional attendees for better availability matching
- **Buffer Context**: Request buffer time when travel or preparation is needed between meetings

Executive calendars are fragmented across multiple stakeholders, projects, and time zones. Scheduling conflicts, inefficient meeting patterns, and lack of deep work time significantly impact productivity. Finding common time for key decisions becomes a bottleneck.

**Solution Workflow:**
1. **Calendar Analysis**: Use `ListCalendarView` to analyze executive's meeting patterns and identify optimization opportunities

3. **Conflict Resolution**: Detect overlapping meetings and suggest reschedules using `UpdateEvent` + `CreateEvent`
4. **Deep Work Protection**: Automatically block focus time by creating recurring "No Meeting" blocks using `CreateEvent`
5. **Stakeholder Coordination**: Use `GetUserDateAndTimeZoneSettings` to align with each stakeholder's preferences
6. **Automated Reminders**: Schedule pre-meeting prep calls and post-meeting debrief sessions using `CreateEvent` with recurring patterns
7. **Meeting Consolidation**: Identify back-to-back meetings and consolidate or combine related topics

**Key Benefits:**
- ⏱️ Reduces scheduling time
- 🎯 Ensures critical decisions happen with full stakeholder alignment
- 💡 Recovers 5–10 hours per week of focused deep work time
- 🌍 Simplifies global team coordination across time zones
- 📈 Improves executive responsiveness and decision velocity

**Tools Used:**
`ListCalendarView`, `FindMeetingTimes`, `UpdateEvent`, `CreateEvent`, `GetUserDateAndTimeZoneSettings`, `CancelEvent`, `AcceptEvent`

## 🔒 Security & Governance Considerations

When deploying Work IQ Calendar MCP Server in enterprise environments, ensure:

- **Access Control**: Implement role-based access control (RBAC) so agents only access and modify calendars they're authorized to manage
- **Delegation Transparency**: Ensure all calendar modifications made on behalf of users are logged and auditable
- **Timezone Accuracy**: Validate timezone handling to prevent scheduling errors across distributed teams
- **Attendee Privacy**: Respect calendar sharing policies and only expose availability data that users have authorized
- **Meeting Link Security**: Ensure Teams meeting links and sensitive meeting details are protected and not exposed unnecessarily
- **Audit Logging**: Enable comprehensive audit logging for all calendar operations (who created/modified/canceled meetings, when, and from where)
- **Consent & Transparency**: Ensure users consent to AI-powered calendar analysis and meeting suggestions
- **Calendar Sharing Policies**: Enforce organizational policies regarding shared calendars and resource booking
- **Data Retention**: Align calendar data retention with corporate and legal retention policies
- **Conflict Resolution**: Implement safeguards to prevent scheduling conflicts and ensure optimal resource utilization
