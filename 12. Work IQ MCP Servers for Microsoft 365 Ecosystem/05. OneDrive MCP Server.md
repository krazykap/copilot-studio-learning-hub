# 05. OneDrive MCP Server - Enabling Personal File Intelligence & Automation in Work IQ
OneDrive MCP Server is a powerful component of the Work IQ ecosystem that enables personal file intelligence and automation for Microsoft 365 users. By integrating with OneDrive, the MCP Server can analyze user files, extract insights, and automate tasks to enhance productivity and streamline workflows.

## 🚀 Why OneDrive MCP Server Matters
While SharePoint MCP Server focuses on team collaboration and shared content, OneDrive MCP Server is designed to empower individual users by providing personalized insights and automation based on their personal files. This allows users to:
- Personalize their Work IQ experience by leveraging insights from their OneDrive files.
- Working documents, spreadsheets, and presentations to extract relevant information and automate tasks.
- Ad-hoc data analysis and reporting based on personal file content.
- Early-stage insights and recommendations for improving productivity based on file usage patterns.

However, it's important to note that OneDrive MCP Server is not intended for large-scale organizational insights or team collaboration scenarios. It is focused on enhancing the individual user experience by providing personalized intelligence and automation based on their personal files, where user struggles with:
- Finding relevant information across their personal files.
- Extracting insights from documents, spreadsheets, and presentations.
- MAnaging versions and updates of personal files.
- Reusing content from personal files in new contexts.

## ⚙️ Core Capabilities
### File Discovery & Search
File Discovery & Search allows users to quickly find relevant information across their personal files in OneDrive. By leveraging AI and natural language processing, users can search for specific content, keywords, or topics within their documents, spreadsheets, and presentations. This capability helps users save time and effort by providing quick access to the information they need without having to manually browse through their files.
It searches files by:
- Content: Search for specific keywords, phrases, or topics within the content of documents, spreadsheets, and presentations.
- Name: Search for files based on their names, titles, or metadata.
- Filters: Apply filters such as file type, date modified, or author to narrow down search results and find relevant files more efficiently.
- File Types: Search across different file types, including Word documents, Excel spreadsheets, PowerPoint presentations, and more.

### File Insights & Analysis
File Insights & Analysis provides users with valuable insights and analysis based on the content of their personal files. By analyzing the content, structure, and usage patterns of documents, spreadsheets, and presentations, users can gain a deeper understanding of their files and identify trends, patterns, and opportunities for improvement. This capability helps users make informed decisions and optimize their workflows based on the insights derived from their personal files.
It provides insights such as:
- Content Analysis: Analyze the content of documents, spreadsheets, and presentations to extract key information, identify trends, and gain insights into file usage patterns.
- Key Metrics: Provide key metrics and statistics about file usage, such as frequency of access, collaboration patterns, and content relevance.

### Content Automation & Recommendations
Content Automation & Recommendations enables users to automate file-related tasks and receive personalized recommendations based on the content of their personal files. By leveraging AI and machine learning, users can automate tasks such as organizing, tagging, and versioning their files, as well as receive recommendations for improving productivity and optimizing workflows based on the insights derived from their personal files.
It provides automation and recommendations such as:
- File Organization: Automatically organize files based on content, metadata, or usage patterns to improve file management and accessibility.
- Auto-Tagging: Automatically tag files with relevant keywords or categories based on their content to enhance searchability and organization.
- Append & Reuse: Automatically suggest relevant content from existing files to reuse in new documents, spreadsheets, or presentations.

## 🧰 MCP Tool Definitions

The OneDrive MCP Server provides a comprehensive set of tools organized into functional categories for personal file intelligence and automation:

### 🔍 File Discovery & Search

| Tool | Description |
|---|---|
| `findFileOrFolderInMyDrive` | Find a file or folder in the user's OneDrive by passing a search query. More efficient than searching across all accessible files if you know the file is in personal OneDrive. Supports keyword and semantic search. |
| `getOnedrive` | Get information about the user's OneDrive including drive metadata, total quota, used space, and owner information. Returns overall OneDrive statistics. |
| `getFolderChildrenInMyOnedrive` | Enumerate files and folders contained within a specified parent folder in OneDrive. Returns top 20 items including file/folder metadata. Useful for browsing folder contents. |

### 📂 File & Folder Management

| Tool | Description |
|---|---|
| `createFolderInMyOnedrive` | Create a new folder in the user's OneDrive as a child of a specified parent folder. Returns newly created folder metadata. Supports nested folder structures. |
| `renameFileOrFolderInMyOnedrive` | Rename a file or folder in OneDrive. New name must comply with OneDrive naming conventions. Returns updated item metadata. |
| `moveSmallFileInMyOnedrive` | Move a file (less than 5MB) to another folder within OneDrive. Only supports moves within personal OneDrive, not across drives. Returns updated file metadata. |
| `deleteFileOrFolderInMyOnedrive` | Delete a file or folder from OneDrive. Removes the item and all contents if it's a folder. Returns confirmation of deletion. |

### 📝 File Content Operations

| Tool | Description |
|---|---|
| `createSmallTextFileInMyOnedrive` | Create or upload a text file (less than 5MB) to OneDrive. Can be uploaded to a specific folder or root if not specified. Useful for creating notes, documents, or configuration files. |
| `readSmallTextFileFromMyOnedrive` | Read/download a text file from OneDrive. Requires file ID (driveItemId). Returns file content as text. Useful for retrieving document content for analysis. |

### 📊 File Metadata & Properties

| Tool | Description |
|---|---|
| `getFileOrFolderMetadataInMyOnedrive` | Get metadata of a file or folder from OneDrive. Returns detailed information including name, size, type, created/modified dates, and access information. |
| `getFileOrFolderMetadataByUrl` | Get metadata of a file or folder from a sharing URL. Only works for users with existing explicit permissions to access the file. Returns metadata without full file content. |

### 🔗 Sharing & Security

| Tool | Description |
|---|---|
| `shareFileOrFolderInMyOnedrive` | Send a sharing invitation to grant read/write permissions on a file or folder in OneDrive. Supports assigning roles (read, edit) and notifying recipients. Can share with individuals or groups. |
| `setSensitivityLabelOnFileInMyOnedrive` | Set the sensitivity/classification label on a file in OneDrive. Applies data protection and compliance labels to files based on organizational policies. Ensures proper data classification. |

## 🔌 Integration with Microsoft Graph
OneDrive MCP Server integrates with Microsoft Graph API to access and manage user files in OneDrive. By leveraging Microsoft Graph, the MCP Server can perform a wide range of operations on user files, including searching, reading, creating, updating, and deleting files and folders. This integration allows the MCP Server to provide personalized file intelligence and automation capabilities based on the content and metadata of user files in OneDrive.

### 📍 Key Graph API Endpoints
| **MCP Tool** | **HTTP Method** | **Graph API End Point** | **Description** |
|---|---|---|---|
| **getDriveRoot** | `GET` | `/me/drive/root` | Retrieve the root folder of the user's primary OneDrive. |
| **listDriveChildren** | `GET` | `/me/drive/root/children` | List all files and folders located in the root directory. |
| **getFolderChildren** | `GET` | `/drives/{driveId}/items/{folderId}/children` | Enumerate items inside a specific folder using its ID. |
| **getItemMetadata** | `GET` | `/me/drive/items/{itemId}` | Get properties (size, name, webUrl, facets) for a file/folder. |
| **getItemByPath** | `GET` | `/me/drive/root:/{path}` | Access a specific file or folder using its relative file path. |
| **searchDrive** | `GET` | `/me/drive/search(q='{query}')` | Search for files and folders across the entire drive. |
| **createFolder** | `POST` | `/me/drive/items/{parentId}/children` | Create a new folder under a specified parent directory. |
| **uploadSmallFile** | `PUT` | `/me/drive/items/{parentId}:/{name}:/content` | Upload a file (up to 4MB) directly via binary stream. |
| **createUploadSession** | `POST` | `/me/drive/items/{itemId}/createUploadSession` | Start a resumable session for large file uploads (>4MB). |
| **downloadFile** | `GET` | `/me/drive/items/{itemId}/content` | Download the raw binary content of a specific file. |
| **updateItem** | `PATCH` | `/me/drive/items/{itemId}` | Update metadata or rename a file/folder. |
| **moveItem** | `PATCH` | `/me/drive/items/{itemId}` | Move an item by updating its `parentReference`. |
| **copyItem** | `POST` | `/me/drive/items/{itemId}/copy` | Copy a file or folder (asynchronous operation). |
| **deleteItem** | `DELETE` | `/me/drive/items/{itemId}` | Move a file or folder to the recycle bin. |
| **listRecent** | `GET` | `/me/drive/recent` | List files recently accessed by the signed-in user. |
| **listSharedWithMe** | `GET` | `/me/drive/sharedWithMe` | List files from other users that are shared with the current user. |
| **createSharingLink** | `POST` | `/me/drive/items/{itemId}/createLink` | Generate a sharing URL (view, edit, or embed). |
| **getPreview** | `POST` | `/me/drive/items/{itemId}/getPreview` | Get a short-lived URL to render a file preview in a browser. |
| **listVersions** | `GET` | `/me/drive/items/{itemId}/versions` | Retrieve the version history of a file. |
| **restoreVersion** | `POST` | `/me/drive/items/{itemId}/versions/{vId}/restoreVersion` | Roll back a file to a specific previous version. |

### 🔐 Required Scopes
OneDrive operations require specific Microsoft Entra ID (formerly Azure AD) permissions. Ensure your application is registered with the following scopes:
- **Files.Read**: Allows the app to read the signed-in user's files.
- **Files.ReadWrite**: Allows the app to read, create, update, and delete the user's files.
- **Files.Read.All**: Allows the app to read all files the user can access (including those shared by others).
- **Files.ReadWrite.All**: Allows the app to read, create, update, and delete all files the user can access.
- **Sites.Read.All**: Required if accessing files within SharePoint-backed OneDrive for Business.

## 🧩 Potential Enterprise Use Cases
### 1️⃣ Personal Productivity & "Draft-to-Publish" Workflow
**Problem Statement:** Employees spend hours manually moving documents from private drafts to team areas. Versioning is often lost during the transition, and "Final_v2_REAL_final.docx" naming conventions create confusion.

**Solution Workflow:**
- **Draft Scaffolding:** Use `createFolder` to automatically set up a standard "Working Drafts" structure for every new project.
- **Intelligent Retrieval:** Use `searchDrive` with content-based queries to find that "one specific chart" buried in an old private spreadsheet.
- **Auto-Naming:** Use `updateItem` to programmatically rename files based on standardized patterns (e.g., YYYY-MM-DD_Project_Title) to eliminate naming chaos.
- **One-Click Publishing:** Once a draft is ready, use `moveItem` or `copyItem` to transition the file from the user's private OneDrive to a public SharePoint Document Library.
- **Format Conversion:** Use the MCP to trigger a "Save as PDF" via `uploadSmallFile` before sharing a final version externally.

**Key Benefits:**
- 🚀 Accelerates the transition from individual ideation to team execution.
- 🧹 Automates file hygiene and naming conventions.
- 📉 Reduces storage bloat by identifying and moving stale drafts.

**Tools Used:** `createFolder`, `searchDrive`, `updateItem`, `moveItem`, `copyItem`, `uploadSmallFile`

### 2️⃣ Secure External Collaboration & Ghosting Prevention
**Problem Statement:** When sharing files with external vendors via OneDrive, permissions are often left "open" indefinitely. Additionally, when an employee leaves the company, their "Shared with Me" links often break, causing work stoppages.

**Solution Workflow:**
- **Time-Bound Sharing:** Use `createSharingLink` with specific scopes (view/edit) and track them in a central "Audit List."
- **Access Review:** Periodically run `getItemMetadata` on shared folders to identify which external users still have active access.
- **Permission Cleanup:** Use `deleteItem` (on permissions/links) to programmatically revoke access to vendors once a contract date has passed.
- **Ghost Link Recovery:** Use `listSharedWithMe` to identify critical project files owned by deactivated accounts and move them to a managed department drive before they are deleted.
- **Secure Previewing:** Use `getPreview` to let users view sensitive files in a secure browser window without allowing them to download a local copy.

**Key Benefits:**
- 🔐 Hardens security by ensuring no "permanent" backdoors exist for ex-vendors.
- 🛡️ Prevents data loss during employee offboarding.
- 📊 Provides visibility into the organization’s external data footprint.

Tools Used: `createSharingLink`, `getItemMetadata`, `listSharedWithMe`, `getPreview`, `deleteItem`.
